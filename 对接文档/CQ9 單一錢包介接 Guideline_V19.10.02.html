CQ9 單一錢包介接 Guideline

  • 貴方實作錢包文件
  • 我方遊戲串接文件

怎樣才能開始營運

  1. 請先於 [測試環境] 建立錢包接口，並且提供錢包接口網址給我們， 我們會建立 [代理測試帳號] 給貴方 測試單一錢包對接
  2. 確認接口實作、與我方對接沒有問題，請提供玩家測試帳號供我方測試，若貴方有使用CNY以外的幣別，再請提供該幣別的子代理測試帳號，與玩家測試帳號供我方測試
  3. 我方會安排人員進行對貴方的單一錢包接口測試，測試通過後會交接給我方QA人員做後續測試，都通過後才會發正式環境申請表單給貴司
  4. 測試環境確認完成，可於正式環境建立單一錢包代理， 並請提供 [正式環境] 的錢包網址，我方這邊要作權限開通
  5. 正式環境開通完成，請貴方測試，若測試正常即可

使用步驟

  1. 請先實作出錢包 API ➜ 貴方實作錢包文件

  2. 當您實作完錢包API後，請提供我們您實作的錢包API URL，才能幫您建立 [測試環境] 後台帳號。

  3. 獲取測試環境後台帳號，就可以接下來調用我們的接口。
    ※我方建議貴司實作完錢包 API 接口後，請先通知我司進行接口測試，測試通過後再調用我方接口，否則遊玩我方遊戲時可能會遇到問題

  4. 請使用您的玩家帳號 account 進行 Player取得遊戲連結

  5. 當您的玩家在玩CQ9遊戲的同時，我們就會呼叫您所實作的錢包API

  6. 結束遊戲的時候，請使用 玩家登出

  7. 或許您還需要其他API接口，那麼請參考我們的文檔： 我方遊戲串接文件

無縫錢包（單一錢包）交易接口

API重點

  1. 我方呼叫貴司API時，會在Header帶入wtoken， 此token由貴司提供，給貴司驗證是否為我方呼叫API。
  2. 所有api回應的錢包幣別，請依據代理幣別填寫，例如 代理的幣別為CNY，此欄位也應同CNY
  3. mtcode為唯一不重複的值，請貴方務必確認其唯一性
  4. 使用的時間格式皆為RFC3339 EX: 2002-10-02T10:00:00.123456789-04:00(請至少支援到毫秒第9位數)
  5. 執行交易時，請務必紀錄下交易前後的金額，金額必須支援16+4小數點。
  6. 收到的參數，請使用Decode
  7. API回應的Content-Type為application/json
  8. 當API呼叫方法為GET時，URL後方使用變數意思等於URL後方直接帶入玩家名稱
    舉例：若欲取得玩家"ABC"的餘額:
    /transaction/balance/:account與/transaction/balance/123的取得方法是一樣的
  9. 交易的紀錄格式如下

_id string 交易紀錄編號
action string 該交易的動作 如 bet, endround, debit, credit, rollin, rollout
target.account string 使用者帳號
status.createtime string 交易開始時間

API 規範

  1. 每次使用API必定會回傳 200 狀態碼，如有其他狀態碼則表示服務不通，或是服務路徑有錯

  2. 時間格式為 RFC3339 EX: 2002-10-02T10:00:00.123456789-04:00(請至少支援到毫秒第9位數)

  3. 每個接口若有用到時區，時間時區為 UTC-4 美東時間 ※很重要

  4. mtcode 交易代碼為自行產生的唯一代碼(不可含特殊字元)，建議可使用UUID

  5. 玩家結算報表、代理結算報表API內若有包含開始時間 (starttime) 與結束時間 (endtime) 參數,該時間皆包含在查詢範圍 。
    例如：
    i. 查詢一日的資料
    設定 開始時間 (starttime) 為 2017-05-15T00:00:00-04:00
    設定 結束時間 (endtime) 為 2017-05-15T23:59:59.999-04:00
    所查詢到的資料範圍會是當天一整天的資料
    ※endtime查詢時請加上.999確保查詢資料完整
    ii. 查詢跨日資料
    設定 開始時間 (starttime) 為 2017-05-14T01:00:00-04:00
    設定 結束時間 (endtime) 為 2017-05-15T00:59:59.999-04:00
    所查詢到的資料範圍會是跨日資料

    除以上兩支API外，其餘API皆使用整點時間進行查詢且均支援查詢一日和跨日的資料
    ※注意有些API限制跨日最多一日 ex.錢包歷程

  6. 測試環境不需要添加白名單；正式環境需要添加白名單

  7. 損益金額計算公式
    玩家損益金額(從玩家角度來計算玩家損益)：
    牌桌類：win-rake
    非牌桌類：win-bet

    代理損益金額(從代理角度來計算代理損益)：
    牌桌類：rake-win
    非牌桌類：bet-win

  8. 當API呼叫方法為GET時，URL後方使用變數意思等於URL後方直接帶入玩家名稱
    舉例：若欲查詢玩家"ABC"的餘額:
    /gameboy/player/balance/:account與/gameboy/player/balance/ABC的查詢方法是一樣的
